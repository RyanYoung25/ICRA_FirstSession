MAESTOR

Machine And Environment Simulation Translation Over Ros

Programming HUBO is hard


  • Original Code: 12,000 lines of C++ in one file
  • Took weeks to familiarize yourself with it
  • The original code was not very flexible
  • Running on Windows XP

Software Rewrite Goals:


  • Make the HUBO easier to program
  • Make the development of gestures safer
  • Make integrating sensors simpler

Where did MAESTOR come from?

  • MAESTRO: MAchine and Environment Software Translation to ROs
    • Second Iteration
    • Needed work on the ease of use and flexibility
  • MAESTOR: Machine And Environment Simulation Translation Over Ros
    • Third and final iteration
    • The software was completely restructured to be more flexible and easier to use.

How does MAESTOR make it easier to program?


  • Simple python API
  • Movements boil down to end position joint angles
  • Students with no programming experience have had success in quickly creating complex gestures.

How does MAESTOR make development safer?


  • OpenRAVE simulation of HUBO allows for off robot testing and development of gestures.
  • Gestures can be run in both simulation and on the physical robot without changing anything.

How does MAESTOR help with sensor integration?


  • Built on top of ROS, gives access to hundreds of ROS packages
  • MAESTOR communication works with both python and C++, can use either language to access sensors and command HUBO

Interacting with MAESTOR


  • The MAESTOR console
  • The MAESTOR python object
  • Any ROS node through ROS service calls

Time for the Hands On portion

Writing a gesture


  • HUBO thinks of motions as joint angles
  • Web-tool to help find appropriate joint angles:
    Hubo-in-the-browser
  • wmhilton.com/hubo-js

Creating your first script



  • hubo@hubo:~$ cd git/maestor/scripts/

  • hubo@hubo:~$ touch workshop.py

  • hubo@hubo:~$ gedit workshop.py

Starting MAESTOR



MAESTOR has three modes of operation:

  1. real - The physical hardware mode
  2. sim - The simulation mode with physics enabled
  3. sim-real - The simulation with physics mode turned off.
    This gives real time motion visualization


hubo@hubo:~$maestor sim-real

The MAESTOR Console

After starting MAESTOR, to run commands and initialize the robot we need to use the console. To start the MAESTOR console run the command:


hubo@hubo:~$maestor console

Start Up


  • >>> initRobot("")

  • >>> command("HomeAll", "")

  • >>> command("EnableAll", "")

The robot is ready to move!

Running your script!


MAESTOR provides a convenient way of running script from anywhere if they are in the /maestor/scripts/ directory and end with .py


hubo@hubo:~$ maestor script workshop
